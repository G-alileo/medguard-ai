Retrieval-Augmented Generation (RAG)

MedGuard combines structured database queries with semantic vector search to provide comprehensive drug safety analysis.

Structured Data

MySQL database stores specific drug interactions, dosages, contraindications from FDA databases

Vector Embeddings

ChromaDB uses BGE-base embeddings to semantically search 500k+ medical documents for contextual information

Information Fusion

DeepSeek LLM combines structured facts with retrieved context to generate personalized explanations

Data Sources

FDA Adverse Events

Real-world adverse event reports from the FDA FAERS database

FDA Drug Labels

Official drug labeling information including indications and warnings

SIDER Dataset

Side effect information extracted from drug package inserts

Technical Stack

Django (Python 3.13)
MySQL + ChromaDB
BGE Embeddings
DeepSeek API

AI-Powered Decision Flow

Step 1: Input Normalization

Drug names and symptoms are standardized using fuzzy matching and brand-to-generic mapping (handles misspellings like "ibrufen" → "ibuprofen")

Step 2: Structured Database Query

MySQL database provides known drug interactions, FDA-approved indications, contraindications, and dosing guidelines

Step 3: Vector Semantic Search

ChromaDB retrieves relevant context from 500k+ medical documents using BGE embeddings - finds nuanced interactions and case studies

Step 4: Risk Calculation

Weighted algorithm combines interaction severity, side effect overlap, treatment appropriateness into a 0-100 risk score

Step 5: RAG Synthesis

DeepSeek LLM combines structured data + retrieved context to generate personalized, explainable safety recommendations

Step 6: Alternative Discovery

System suggests safer therapeutic alternatives based on your specific symptom profile and existing medications
